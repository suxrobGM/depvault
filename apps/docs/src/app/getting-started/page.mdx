# Getting Started

## Create Your Account

DepVault supports two registration methods: email/password or GitHub OAuth.

### Email and Password

1. Navigate to the login page and click **Register**.
2. Fill in your name, email, and a password (minimum 8 characters, at least 1 uppercase letter and 1 number).
3. Click **Create Account**, then verify your email via the link sent to your inbox.

### GitHub OAuth

Click **Continue with GitHub** on the login page and authorize DepVault. Your account is created automatically from your GitHub profile.

---

## Create Your First Project

1. From the **Dashboard**, click **+ New Project**.
2. Enter a project name and optional description.
3. Click **Create**.

Your project appears as a card on the dashboard, ready for dependency scans and config/secret file management.

---

## Run Your First Dependency Scan

### Web UI

1. Open your project and go to the **Analysis** tab.
2. Click **New Analysis** and upload a dependency file (e.g., `package.json`, `requirements.txt`, `Cargo.toml`).
3. Click **Analyze** to view results with version status and vulnerability data.

### CLI

Install the CLI and run a scan:

```bash
# Linux / macOS
curl -fsSL https://get.depvault.com | bash

# Windows (PowerShell)
irm https://get.depvault.com | iex
```

## Push Your Config & Secret Files

Push your repository's config and secret files to DepVault. Each file is encrypted in your browser/CLI and uploaded as a single blob:

```bash
depvault login
depvault push
```

`push` discovers config files (`.env`, `appsettings.json`, …) and secret files (`.pem`, `.key`, `credentials.json`, …), infers each file's app and environment slug from its path and name, and uploads the encrypted blobs. No `--vault` or `--tag` flags needed.

Pull everything back, restored byte-for-byte to its original repo-relative path:

```bash
depvault pull
```

This recreates each file exactly where it was pushed from, recreating directories as needed. See the [Pull & Push](/cli/pull) reference for all options.

---

## Edit a File in the Browser

1. Open your project and go to the repository view.
2. Pick an app in the sidebar and an environment tab, then select a config file.
3. Edit it in the **Form** (key/value table for `.env`) or **Raw** (code editor) view, click **Review & Save** to preview a git-style diff, and save.

Every save snapshots a new encrypted version. See the [Config & Secret Files](/guides/environment-vault) guide for details.
