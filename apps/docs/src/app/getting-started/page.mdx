# Getting Started

## Create Your Account

DepVault supports two registration methods: email/password or GitHub OAuth.

### Email and Password

1. Navigate to the login page and click **Register**.
2. Fill in your name, email, and a password (minimum 8 characters, at least 1 uppercase letter and 1 number).
3. Click **Create Account**, then verify your email via the link sent to your inbox.

### GitHub OAuth

Click **Continue with GitHub** on the login page and authorize DepVault. Your account is created automatically from your GitHub profile.

---

## Create Your First Project

1. From the **Dashboard**, click **+ New Project**.
2. Enter a project name and optional description.
3. Click **Create**.

Your project appears as a card on the dashboard, ready for dependency scans and secret management.

---

## Run Your First Dependency Scan

### Web UI

1. Open your project and go to the **Analysis** tab.
2. Click **New Analysis** and upload a dependency file (e.g., `package.json`, `requirements.txt`, `Cargo.toml`).
3. Click **Analyze** to view results with version status and vulnerability data.

### CLI

Install the CLI and run a scan:

```bash
# Linux / macOS
curl -fsSL https://get.depvault.com | bash

# Windows (PowerShell)
irm https://get.depvault.com | iex
```

---

## Set Up Your Environment

Scan your repository to discover environment files and push them to the vault:

```bash
depvault login
depvault scan
```

The scan command detects `.env` files, maps each directory to a vault group, and uploads variables. Each vault group stores the directory path so pull knows where to write files.

Pull environment variables and secret files back to the correct folders:

```bash
depvault pull
```

This writes `.env` files to each vault group's configured directory and downloads secret files to `.secrets/` folders. See the [Pull & Push](/cli/pull) reference for all options.

---

## Store Your First Secret

1. Open your project and go to the **Vault** tab.
2. Select an environment (**DEVELOPMENT**, **STAGING**, **PRODUCTION**, or **GLOBAL**).
3. Click **Add Variable**, enter the key (e.g., `DATABASE_URL`) and value, then click **Save**.

Values are encrypted with AES-256-GCM immediately upon storage.

To bulk-import from an existing `.env` file, use the **Import** button in the Vault tab or `depvault push env` from the CLI. See the [Environment Vault](/guides/environment-vault) guide for details.
