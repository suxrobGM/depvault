# Team Management

Invite members, assign roles, and manage access permissions for your projects.

---

## Roles

| Permission | Owner | Editor | Viewer |
| --- | --- | --- | --- |
| View dashboard and analysis results | Yes | Yes | Yes |
| Run dependency analysis | Yes | Yes | No |
| View config/secret file list | Yes | Yes | Yes |
| Decrypt, edit, and download files | Yes | Yes | No |
| Push and upload files | Yes | Yes | No |
| Generate sharing links | Yes | Yes | No |
| Manage scan results | Yes | Yes | No |
| Invite/remove members | Yes | No | No |
| Change member roles | Yes | No | No |
| Modify project settings | Yes | No | No |
| Delete project / transfer ownership | Yes | No | No |

- **Owner** — Full administrative control. One per project. Automatically assigned to the project creator.
- **Editor** — Full contributor access (read/write data, run analyses). No administrative actions.
- **Viewer** — Read-only. Cannot decrypt file contents, download files, or modify data. Useful for stakeholders and auditors.

---

## Inviting Members

Open the **Members** tab, click **Invite Member**, enter the recipient's email, select a role (Editor or Viewer), and send. Only the Owner can invite. The Owner role can only be obtained through ownership transfer.

---

## Changing Roles and Removing Members

From the **Members** tab, use the role dropdown to switch a member between Editor and Viewer (Owner only). Click the remove button to revoke access immediately — a new invitation is required to restore it.

---

## Transferring Ownership

Open the **Members** tab, click **Transfer Ownership**, and select an existing member. The selected member becomes Owner; the previous owner is demoted to Editor. This is immediate and cannot be undone.

---

## Project Deletion

Only the Owner can delete a project. Navigate to **Settings** > **Danger Zone** > **Delete Project** and confirm by typing the project name. This removes all data (analyses, config and secret files, scan results) and is irreversible.
