# API Reference

Complete reference for the DepVault REST API. All endpoints are organized under the `/api` prefix and follow REST conventions with consistent request/response shapes.

## Base URL

| Environment | Base URL |
|---|---|
| Production | `https://depvault.com/api` |
| Local development | `http://localhost:4000/api` |

All endpoint paths in this documentation are relative to the base URL. For example, `POST /auth/login` refers to `https://depvault.com/api/auth/login` in production.

## Content Types

- **Request and response bodies** use `application/json` by default.
- **File uploads** (dependency file analysis, env import) use `multipart/form-data` with a maximum file size of **25 MB**.

## Interactive Documentation

A live Swagger UI is available at [`/api/swagger`](http://localhost:4000/api/swagger) for exploring and testing endpoints directly in the browser. The raw OpenAPI specification can be downloaded from `/api/swagger/json` for client generation.

## Rate Limiting

Rate limits are applied per client IP using a fixed-window counter. Every response includes these headers:

| Header | Description |
|---|---|
| `x-ratelimit-limit` | Maximum requests allowed in the window |
| `x-ratelimit-remaining` | Requests remaining in the current window |
| `x-ratelimit-reset` | Unix timestamp (seconds) when the window resets |
| `retry-after` | Seconds until the next request is allowed (only on 429 responses) |

Auth-related endpoints have stricter limits. See the [Authentication](/api/authentication) page for per-endpoint limits.

## Authentication

Most endpoints require a valid JWT. The API uses httpOnly cookies for browser clients and `Authorization: Bearer` headers for CI/programmatic access. See the [Authentication](/api/authentication) page for the full flow.

## API Groups

The following table lists every endpoint group available in the DepVault API.

| Group | Prefix | Description |
|---|---|---|
| **Auth** | `/auth` | Registration, login, token refresh, logout, password reset, email verification, and GitHub OAuth |
| **Users** | `/users` | User profile management, avatar upload, and account settings |
| **Projects** | `/projects` | Create, list, update, and delete projects |
| **Members** | `/projects/:id/members` | Invite, list, update roles, and remove project members |
| **Analyses** | `/analyses` | Upload dependency files for parsing, view analysis results and dependency trees |
| **Environments** | `/environments` | Create and manage environments (e.g., development, staging, production) within a project |
| **Env Variables** | `/env-variables` | CRUD operations for encrypted environment variables within an environment |
| **Env Diff** | `/env-diff` | Compare environment variables across two environments side by side |
| **Env Import/Export** | `/env-io` | Import variables from `.env` files or export environments to various formats |
| **Env Bundle** | `/env-bundle` | Download all variables for an environment as a single `.env` bundle |
| **Env Templates** | `/env-templates` | Define and manage onboarding templates with required/optional variable definitions |
| **Secret Files** | `/secret-files` | Upload and manage encrypted file attachments (certificates, keystores, config files) |
| **Shared Secrets** | `/shared-secrets` | Generate one-time, time-limited links for securely sharing environment variables or secrets |
| **CI Tokens** | `/ci-tokens` | Create and manage machine tokens for CI/CD pipeline authentication |
| **CI Access** | `/ci-access` | Retrieve environment variables in CI pipelines using a CI token |
| **Secret Scanning** | `/secret-scans` | Run and view results of secret scanning across dependency and config files |
| **Scan Patterns** | `/scan-patterns` | Manage custom regex patterns used by the secret scanner |
| **License Rules** | `/license-rules` | Configure allowed, denied, and flagged open-source license policies |
| **Audit Log** | `/audit-log` | View a chronological log of all actions taken within a project |
| **Activity** | `/activity` | View recent activity feed for the authenticated user across all projects |
| **Notifications** | `/notifications` | List, mark as read, and manage in-app notifications |
| **GitHub** | `/github` | List repositories and import projects from a linked GitHub account |
| **Convert** | `/convert` | Convert between configuration file formats (e.g., `.env` to `appsettings.json`, `docker-compose.yml` to `.env`) |

## Pagination

List endpoints return paginated results using a consistent shape:

```json
{
  "items": [ ... ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 57,
    "totalPages": 3
  }
}
```

Pass `page` and `limit` as query parameters to control pagination. The default page size is 20.
