# Secret Files

The `depvault secrets` command group lists encrypted secret file metadata stored for a project. Secret files (certificates, keys, credentials) are stored as whole client-encrypted blobs — see [Secret Files](/guides/secret-files).

---

## secrets list

List secret file metadata for the active project.

```bash
depvault secrets list
```

| Flag | Description | Default |
| --- | --- | --- |
| `--project` | Project ID. Uses active project if omitted. | Active project |
| `--environment` | Filter by environment slug (e.g. `base`, `dev`, `prod`). | All |
| `--output` | Output format: `table`, `json`. | `table` |

The table shows each file's path, environment slug, owning app, MIME type, size, and last-updated date.

### Examples

List all secret files:

```bash
depvault secrets list
```

List production secret files as JSON:

```bash
depvault secrets list --environment prod --output json
```

---

To upload secret files, run [`depvault push`](/cli/pull#push). To restore them byte-for-byte to their original paths, run [`depvault pull`](/cli/pull).
