# Convert

Convert configuration files between formats such as `.env`, JSON, YAML, and TOML using the CLI.

---

## Overview

The `depvault convert` command transforms configuration files from one format to another without connecting to the DepVault API. It runs entirely offline and does not require authentication, making it a standalone utility you can use regardless of whether you have a DepVault account.

---

## depvault convert

Convert a configuration file from one format to another.

```bash
depvault convert --file <path> --from <format> --to <format> [flags]
```

### Flags

| Flag       | Description                                                     | Required |
|------------|-----------------------------------------------------------------|----------|
| `--file`   | Path to the input configuration file.                           | Yes      |
| `--from`   | Source format: `env`, `appsettings.json`, `secrets.yaml`, `config.toml`. | Yes |
| `--to`     | Target format: `env`, `appsettings.json`, `secrets.yaml`, `config.toml`. | Yes |
| `--output` | File path to write the converted output. Prints to stdout if omitted. | No |

---

## Supported Formats

| Format              | Description                                    | Example extension |
|---------------------|------------------------------------------------|-------------------|
| `env`               | Standard KEY=VALUE `.env` format               | `.env`            |
| `appsettings.json`  | .NET-style nested JSON configuration           | `.json`           |
| `secrets.yaml`      | Flat or nested YAML key-value pairs            | `.yaml`, `.yml`   |
| `config.toml`       | TOML key-value configuration                   | `.toml`           |

All format combinations are supported. You can convert from any format to any other format.

---

## Examples

### .env to appsettings.json

```bash
depvault convert --file .env --from env --to appsettings.json --output appsettings.json
```

Input (`.env`):

```
DATABASE_URL=postgresql://localhost:5432/mydb
REDIS_URL=redis://localhost:6379
API_KEY=sk-abc123def456
DEBUG=true
```

Output (`appsettings.json`):

```json
{
  "DATABASE_URL": "postgresql://localhost:5432/mydb",
  "REDIS_URL": "redis://localhost:6379",
  "API_KEY": "sk-abc123def456",
  "DEBUG": "true"
}
```

### appsettings.json to YAML

```bash
depvault convert --file appsettings.json --from appsettings.json --to secrets.yaml
```

Output (stdout):

```yaml
DATABASE_URL: postgresql://localhost:5432/mydb
REDIS_URL: redis://localhost:6379
API_KEY: sk-abc123def456
DEBUG: "true"
```

### YAML to TOML

```bash
depvault convert --file secrets.yaml --from secrets.yaml --to config.toml --output config.toml
```

Output (`config.toml`):

```toml
DATABASE_URL = "postgresql://localhost:5432/mydb"
REDIS_URL = "redis://localhost:6379"
API_KEY = "sk-abc123def456"
DEBUG = "true"
```

### TOML to .env

```bash
depvault convert --file config.toml --from config.toml --to env --output .env
```

Output (`.env`):

```
DATABASE_URL=postgresql://localhost:5432/mydb
REDIS_URL=redis://localhost:6379
API_KEY=sk-abc123def456
DEBUG=true
```

---

## Stdout Output

When `--output` is omitted, the converted content is printed to stdout. This lets you pipe the output to other commands or inspect the result before writing to a file:

```bash
# Preview the conversion
depvault convert --file .env --from env --to secrets.yaml

# Pipe to clipboard (macOS)
depvault convert --file .env --from env --to appsettings.json | pbcopy

# Pipe to another command
depvault convert --file .env --from env --to config.toml | tee config.toml
```

---

## No Authentication Required

Unlike most DepVault CLI commands, `convert` is a fully offline operation. It does not send any data to the DepVault API and works without logging in or setting a `DEPVAULT_TOKEN`. This makes it useful as a general-purpose configuration file converter in any project, even outside of DepVault workflows.

---

## Error Cases

| Error                                        | Cause                                              |
|----------------------------------------------|----------------------------------------------------|
| `Error: File not found: <path>`              | The specified input file does not exist.            |
| `Error: Unsupported format: <value>`         | The `--from` or `--to` format is not recognized.   |
| `Error: Failed to parse <path> as <format>`  | The input file content does not match the specified source format. |
