# Environment

Pull, push, list, and diff environment variables for your projects using the CLI.

---

## Overview

The environment commands let you manage encrypted environment variables stored in DepVault directly from the command line. You can export variables to local files, import from existing configuration files, list stored variables, and compare values across environments.

All environment commands require an active project. Set one with `depvault project select <id>` or pass the `--project` flag.

---

## Supported Formats

The CLI supports four configuration file formats for import and export:

| Format              | File extension       | Description                             |
|---------------------|----------------------|-----------------------------------------|
| `env`               | `.env`               | Standard KEY=VALUE format               |
| `appsettings.json`  | `.json`              | .NET-style nested JSON configuration    |
| `secrets.yaml`      | `.yaml` / `.yml`     | Flat or nested YAML key-value pairs     |
| `config.toml`       | `.toml`              | TOML key-value configuration            |

## depvault env pull

Export environment variables from DepVault to a local file or stdout. Variables are decrypted on the server and delivered in the requested format.

```bash
depvault env pull [flags]
```

### Flags

| Flag              | Description                                                        | Default         |
|-------------------|--------------------------------------------------------------------|-----------------|
| `--project`       | Project ID. Uses active project if omitted.                        | Active project  |
| `--vault-group`   | Vault group ID to pull from.                                       |                 |
| `--environment`   | Target environment: DEVELOPMENT, STAGING, PRODUCTION, or GLOBAL.   | `DEVELOPMENT`   |
| `--format`        | Output format: `env`, `appsettings.json`, `secrets.yaml`, `config.toml`. | `env`     |
| `--output`        | File path to write output. Prints to stdout if omitted.            | stdout          |

### Examples

Pull development variables as a `.env` file:

```bash
depvault env pull --vault-group abc123 --output .env
```

Pull production variables in JSON format:

```bash
depvault env pull --vault-group abc123 --environment PRODUCTION --format appsettings.json --output appsettings.Production.json
```

Pull to stdout for piping:

```bash
depvault env pull --vault-group abc123 --format env | grep DATABASE
```

### Output Formats

**env** (default):

```
DATABASE_URL=postgresql://localhost:5432/mydb
REDIS_URL=redis://localhost:6379
API_KEY=sk-abc123def456
```

**appsettings.json**:

```json
{
  "ConnectionStrings": {
    "Database": "postgresql://localhost:5432/mydb"
  },
  "Redis": {
    "Url": "redis://localhost:6379"
  }
}
```

**secrets.yaml**:

```yaml
DATABASE_URL: postgresql://localhost:5432/mydb
REDIS_URL: redis://localhost:6379
API_KEY: sk-abc123def456
```

**config.toml**:

```toml
DATABASE_URL = "postgresql://localhost:5432/mydb"
REDIS_URL = "redis://localhost:6379"
API_KEY = "sk-abc123def456"
```

## depvault env push

Import environment variables from a local file into DepVault. Existing variables with matching keys are updated; new keys are created.

```bash
depvault env push [flags]
```

### Flags

| Flag              | Description                                                        | Default         |
|-------------------|--------------------------------------------------------------------|-----------------|
| `--vault-group`   | Vault group ID to push to. **Required.**                           |                 |
| `--file`          | Path to the local file to import. **Required.**                    |                 |
| `--project`       | Project ID. Uses active project if omitted.                        | Active project  |
| `--environment`   | Target environment: DEVELOPMENT, STAGING, PRODUCTION, or GLOBAL.   | `DEVELOPMENT`   |
| `--format`        | Input format: `env`, `appsettings.json`, `secrets.yaml`, `config.toml`. | `env`      |

### Examples

Push a `.env` file to development:

```bash
depvault env push --vault-group abc123 --file .env
```

Push a YAML secrets file to production:

```bash
depvault env push --vault-group abc123 --file secrets.yaml --environment PRODUCTION --format secrets.yaml
```

### Output

```
Imported 12 variables to vault group "Backend Secrets" (DEVELOPMENT).
  Created: 8
  Updated: 4
```

## depvault env list

List environment variables stored in DepVault for the active project. Values are masked by default in table output.

```bash
depvault env list [flags]
```

### Flags

| Flag              | Description                                                        | Default         |
|-------------------|--------------------------------------------------------------------|-----------------|
| `--vault-group`   | Filter by vault group ID.                                          |                 |
| `--environment`   | Filter by environment: DEVELOPMENT, STAGING, PRODUCTION, or GLOBAL.|                 |
| `--project`       | Project ID. Uses active project if omitted.                        | Active project  |
| `--output`        | Output format: `table` (default) or `json`.                       | `table`         |

### Example

```bash
depvault env list --vault-group abc123 --environment DEVELOPMENT
```

```
  Key              Value              Environment    Updated
  ─────────────────────────────────────────────────────────────
  DATABASE_URL     ********5432/mydb  DEVELOPMENT    2025-12-01
  REDIS_URL        ********6379       DEVELOPMENT    2025-12-01
  API_KEY          ********ef456      DEVELOPMENT    2025-11-28
  DEBUG            true               DEVELOPMENT    2025-11-20
```

### JSON Output

```bash
depvault env list --vault-group abc123 --output json
```

```json
[
  {
    "key": "DATABASE_URL",
    "value": "postgresql://localhost:5432/mydb",
    "environment": "DEVELOPMENT",
    "updatedAt": "2025-12-01T10:30:00Z"
  }
]
```

> JSON output includes full decrypted values. Use caution when redirecting to files.

## depvault env diff

Compare environment variables between two environments within the same vault group. Shows variables that are added, removed, or changed.

```bash
depvault env diff [flags]
```

### Flags

| Flag              | Description                                                                      | Default         |
|-------------------|----------------------------------------------------------------------------------|-----------------|
| `--vault-group`   | Vault group ID. **Required.**                                                    |                 |
| `--environments`  | Comma-separated pair of environments to compare. **Required.**                   |                 |
| `--project`       | Project ID. Uses active project if omitted.                                      | Active project  |
| `--output`        | Output format: `table` (default) or `json`.                                     | `table`         |

### Example

```bash
depvault env diff --vault-group abc123 --environments DEVELOPMENT,PRODUCTION
```

```
  Key              DEVELOPMENT              PRODUCTION           Status
  ──────────────────────────────────────────────────────────────────────────
  DATABASE_URL     localhost:5432/mydb       rds.amazonaws.com    CHANGED
  REDIS_URL        localhost:6379            elasticache.aws      CHANGED
  DEBUG            true                     (not set)             REMOVED
  SENTRY_DSN       (not set)                https://sentry.io    ADDED
  API_KEY          sk-abc123                sk-abc123             SAME
```

### JSON Output

```bash
depvault env diff --vault-group abc123 --environments DEVELOPMENT,PRODUCTION --output json
```

```json
{
  "left": "DEVELOPMENT",
  "right": "PRODUCTION",
  "differences": [
    {
      "key": "DATABASE_URL",
      "left": "localhost:5432/mydb",
      "right": "rds.amazonaws.com",
      "status": "CHANGED"
    },
    {
      "key": "DEBUG",
      "left": "true",
      "right": null,
      "status": "REMOVED"
    }
  ]
}
```
